# PrivateStater â€” Captcha API Reference

Base path: `https://privatestater.com/oapi/captcha`

All requests require `Authorization: Bearer <apiKey>`.

> **Note**: Token validation can also use the public endpoint `/api/captcha/validate` which does not require an API key. See the Validate section below.

---

## GET `/websites/:websiteId/settings`

Returns the current captcha configuration for a website.

**Path params**:
- `websiteId` (string) â€” the unique website identifier

**Response**:
```json
{
  "data": {
    "settings": {
      "enabled": true,
      "allowLocalhost": false,
      "honeypotEnabled": true,
      "powLevel": "medium"
    }
  }
}
```

### Settings Fields

| Field | Type | Description |
|-------|------|-------------|
| `enabled` | boolean | Whether captcha is active on this website |
| `allowLocalhost` | boolean | Allows widget to load on `localhost` / `127.0.0.1` for local testing |
| `honeypotEnabled` | boolean | Enables the invisible honeypot trap as the first layer of bot detection |
| `powLevel` | string | Proof-of-Work difficulty: `low` (~1s), `medium` (~2s), `high` (~3s) |

---

## PUT `/websites/:websiteId/settings`

Updates captcha settings. Only include fields you want to change â€” all fields are optional.

**Path params**:
- `websiteId` (string)

**Request body**:
```json
{
  "enabled": true,
  "allowLocalhost": false,
  "honeypotEnabled": true,
  "powLevel": "medium"
}
```

### Field Reference

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | boolean | `false` | Activate or deactivate captcha for this website |
| `allowLocalhost` | boolean | `false` | Set `true` during development to allow widget on localhost |
| `honeypotEnabled` | boolean | `true` | Set `false` to skip the honeypot step (not recommended) |
| `powLevel` | string | `"medium"` | `"low"` / `"medium"` / `"high"` â€” higher = harder for bots, slightly slower for users |

**Response**: same shape as GET settings

**Example â€” enable captcha with high PoW**:
```javascript
await fetch(
  `https://privatestater.com/oapi/captcha/websites/${websiteId}/settings`,
  {
    method: 'PUT',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ enabled: true, powLevel: 'high' })
  }
);
```

## GET `/websites/:websiteId/stats`

Returns captcha event statistics for a given time range.

**Query params**:
- `range` (required): `1h` `12h` `1d` `7d` `30d`

**Response**:
```json
{
  "data": {
    "stats": {
      "attempt": 520,
      "success": 488,
      "fail_honeypot": 14,
      "fail_puzzle": 9,
      "fail_pow": 5,
      "fail_expired": 4,
      "totalFail": 32,
      "successRate": 93
    },
    "range": "1d"
  }
}
```

### Stats Fields

| Field | Description |
|-------|-------------|
| `attempt` | Total number of captcha attempts (all outcomes) |
| `success` | Completed the full 3-step verification and received a valid token |
| `fail_honeypot` | Blocked at step 1 â€” triggered the invisible honeypot field (bot indicator) |
| `fail_puzzle` | Failed the visual puzzle challenge at step 2 |
| `fail_pow` | Failed or timed out during Proof-of-Work computation at step 3 |
| `fail_expired` | Submitted a token that had already expired (>5 min) or was already used |
| `totalFail` | Sum of all failure types: `fail_honeypot + fail_puzzle + fail_pow + fail_expired` |
| `successRate` | `Math.round(success / attempt * 100)` â€” percentage of attempts that succeeded |

**Answer format**: "Today your captcha had **520 attempts** with a **93% success rate**. **14** were blocked by honeypot, **9** failed the puzzle."

**Example**:
```javascript
const res = await fetch(
  `https://privatestater.com/oapi/captcha/websites/${websiteId}/stats?range=1d`,
  { headers: { 'Authorization': `Bearer ${apiKey}` } }
);
const { data } = await res.json();
```

## POST `/validate`

Validates a captcha token generated by the client-side widget.

> **Two endpoints are available** â€” choose based on your context:
>
> | Endpoint | Auth required | Recommended when |
> |----------|--------------|-----------------|
> | `https://privatestater.com/api/captcha/validate` | None | Server-side form validation (simplest) |
> | `https://privatestater.com/oapi/captcha/validate` | API key | Already using the Open API in the same request |

**Request body** (identical for both endpoints):
```json
{
  "websiteId": "my-blog",
  "verifyToken": "<token from client>"
}
```

**Success response**:
```json
{ "success": true }
```

**Failure response**:
```json
{ "success": false, "error": "<error_code>" }
```

### Error Codes

| Error Code | Cause | Resolution |
|-----------|-------|-----------|
| `token_invalid_or_expired` | Token expired (>5 min after generation) or already consumed | Show captcha again; do not reuse tokens |
| `website_id_mismatch` | The token was generated for a different website ID | Verify that `websiteId` matches `prstSite` in the client config |
| `captcha_not_enabled` | Captcha is disabled in dashboard settings | Enable captcha at Dashboard â†’ Website â†’ Captcha â†’ Settings |
| `host_mismatch` | The request `Origin` header does not match the registered host | Ensure the server request comes from the correct origin, or check the registered Host |

### Important Rules

- Tokens are **single-use** â€” once validated (success or failure), they cannot be reused
- Tokens expire **5 minutes** after generation on the client side
- Always validate server-side; never trust the token on the client alone
- Reset `captchaToken` to `null` after each submission to force re-completion on the next attempt

**Full server-side example (Node.js/Express)**:
```javascript
app.post('/submit', async (req, res) => {
  const result = await fetch('https://privatestater.com/api/captcha/validate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      websiteId: 'my-blog',
      verifyToken: req.body.captchaToken
    })
  }).then(r => r.json());

  if (!result.success) {
    return res.status(400).json({ error: result.error || 'captcha_failed' });
  }

  // captcha passed â€” proceed with request handling
});
```
